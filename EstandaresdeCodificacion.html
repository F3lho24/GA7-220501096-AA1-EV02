Estándares de Codificación

Buenas prácticas para escribir código limpio y mantenible

¿Qué son los estándares de codificación?

Son conjuntos de reglas y convenciones que definen cómo escribir y organizar el código en un proyecto. Estos estándares ayudan a:

  • Mantener consistencia en el código
  • Facilitar el trabajo en equipo
  • Mejorar la legibilidad y mantenibilidad
  • Reducir errores comunes

Tipos de Estándares

Nomenclatura

  • Variables: camelCase (JavaScript) o snake_case (Python)
  • Clases: PascalCase
  • Constantes: MAYÚSCULAS_CON_GUIONES

Formato

  • Indentación: 2 o 4 espacios (no tabs)
  • Línea máxima: 80-120 caracteres
  • Llaves: Mismo estilo en todo el proyecto

Comentarios

  • Documentación: Usar JSDoc, JavaDoc, etc.
  • Comentarios de línea: Explicar "por qué" no "qué"
  • Eliminar: Código comentado obsoleto

Estructura

  • Organización: MVC, Componentes, etc.
  • Archivos: Nombre descriptivo y extensión correcta
  • Jerarquía: Directorios lógicos

Ejemplos Prácticos

JavaScript

// MAL
function getuserdata(){...}

// BIEN (camelCase)
function getUserData() {...}

// Constantes
const MAX_USERS = 10;

Python

# MAL
def CalculateTotal():...

# BIEN (snake_case)
def calculate_total():...

# Constantes
MAX_CONNECTIONS = 5

Menos Errores
